## Introduction
Streamlit is an **open-source Python framework** that allows you to create **interactive web applications** for data science, machine learning, and general Python projects **without any front-end knowledge**.  

> You write Python scripts, and Streamlit automatically renders them as web apps.

---

## Why Streamlit?
- **Fast prototyping:** Turn Python scripts into apps in minutes.  
- **Interactive:** Use sliders, buttons, inputs, and live updates.  
- **Integration:** Works with Pandas, NumPy, Matplotlib, Plotly, PyTorch, TensorFlow, and more.  
- **Open Source:** Free and community-driven.

---

## Core Concepts
1. **Scripts to Apps:** Every Streamlit app is a Python script (`.py`).  
2. **Reactive:** Streamlit reruns your script from top to bottom whenever user input changes.  
3. **Widgets:** Interactive elements like buttons, sliders, checkboxes, and text inputs.  
4. **State Management:** Control app state across user interactions using `st.session_state`.  

---

## Widgets
Widgets let users interact with your app. Examples:
- `st.button()`
- `st.slider()`
- `st.checkbox()`
- `st.text_input()`
- `st.selectbox()`

> Widgets automatically trigger a rerun of the script.

---

## Data Display
Streamlit supports multiple ways to display data:
- **Text & Markdown:** `st.write()`, `st.markdown()`, `st.text()`
- **Tables & DataFrames:** `st.dataframe()`, `st.table()`
- **Charts:** `st.line_chart()`, `st.bar_chart()`, `st.area_chart()`
- **Custom Plots:** Using Matplotlib, Plotly, Altair, or Seaborn
- **Media:** `st.image()`, `st.audio()`, `st.video()`

---

## Layouts & Pages
- **Columns & Containers:** Use `st.columns()` and `st.container()` for responsive layouts.  
- **Tabs & Expanders:** Organize sections with `st.tabs()` and `st.expander()`.  
- **Pages (Multi-page apps):** Use multiple Python files and `st.experimental_set_page_config()` for navigation.

---

## Advanced Features
- **Caching:** `@st.cache_data` and `@st.cache_resource` to optimize performance.  
- **Session State:** `st.session_state` allows storing variables between reruns.  
- **Custom Components:** Use `streamlit-component-lib` to integrate custom JS/HTML.  
- **Forms:** `st.form()` to batch user inputs before submission.  
- **Theming:** Customize app appearance with `config.toml` or themes.

---

## Best Practices
1. Keep the **script simple and modular**. Split large apps into multiple files.  
2. Use **`st.cache_data`** for expensive computations.  
3. Avoid global variables; prefer **`st.session_state`** for stateful apps.  
4. Structure your UI with **columns, containers, and tabs** for clarity.  
5. Use **requirements.txt** to track dependencies.
